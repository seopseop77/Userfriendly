| 3 | user_input  | t02 prompt, same shape as row 1 |
| 4 | tool_result | `[{type:"tool_result", content:"The single top-level key is **`slideshow`**.", tool_use_id:"…", **cache_control:{…}**}]` |

## Findings

1. **WebFetch `tool_result` content is post-processed by an internal LLM,
   not the raw page envelope.** Both rows show summarised structured
   output ("**Main Heading:**…", "The single top-level key is…")
   rather than the `Web page content:\n---\n…` shape that the
   d1e8ae4 wrapper-prefix entry was added against.
2. **The `Web page content:\n---\n` wrapper prefix (d1e8ae4) was not
   exercised by this round.** Two interpretations:
   - That prefix is for a different code path (e.g. the framework
     synthesises a `role=user` text block containing the raw page when
     `<webfetch_url=…>` triggers a side-call). Not visible from the
     plain `WebFetch` tool invocation by the model.
   - OR the prefix is currently dead in production traffic (no path
     produces it).
   - **Probe-able directly with a follow-up**: query
     `plugin_analytics` historic rows for any `request_jsonb::text
     LIKE '%Web page content:\n---\n%'` — if zero hits, the entry is
     dead. Worth doing in the consolidated report rather than as a
     fresh round.
3. **Framework-typed `tool_result` blocks carry `cache_control`.**
   Both WebFetch tool_result rows have
   `cache_control:{ttl:"1h", type:"ephemeral"}`. ADR-0038 noted the
   observed pattern "every framework-typed block does *not* carry
   cache_control" as a *fragile* signal it was rejected for
   classification — this round provides a concrete counterexample.
   **Not a bug** (the classifier does not use cache_control as a
   routing signal), but a clean piece of evidence that the ADR's
   observation about cache_control distribution was overstated. Worth
   updating the ADR's "Sidecar separation signals" prose, if anyone
   cites that observation later.
4. **`input_tokens=1` on tool_result rows.** Both row 2 and row 4
   report `input_tokens=1` instead of 3. Likely accounting artefact of
   the prompt cache hit covering nearly the entire input — the
   uncached delta is tiny.
